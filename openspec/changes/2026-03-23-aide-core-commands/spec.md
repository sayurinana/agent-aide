# 规范：aide 核心子命令

## ADDED: aide hi 命令

### 场景：在常驻分支查看状态（无未归档任务）

**前置条件**：
- 当前在常驻分支
- tasks/ 目录为空

**操作**：
```bash
aide hi
```

**预期输出**：
```
项目：/absolute/path/to/project
分支：dev (常驻分支)

✓ 当前状态干净
  - 无未归档任务
  - task-now.md 已重置
  - task-now/ 目录不存在
```

### 场景：在常驻分支查看状态（有未归档任务）

**前置条件**：
- 当前在常驻分支
- tasks/ 下有 task-3/ 和 task-5/

**操作**：
```bash
aide hi
```

**预期输出**：
```
项目：/absolute/path/to/project
分支：dev (常驻分支)

未归档任务：

→ 任务 #3
  实现用户认证功能
  最后提交：2026-03-20 15:30:45 +0800 (3天前)

→ 任务 #5
  优化数据库查询性能
  最后提交：2026-03-22 10:15:20 +0800 (6小时前)

提示：使用 'aide go 5' 进入最近活跃的任务
```

### 场景：在任务分支查看状态

**前置条件**：
- 当前在任务分支 task-3

**操作**：
```bash
aide hi
```

**预期输出**：
```
项目：/absolute/path/to/project
分支：task-3 (任务分支)

→ 任务 #3：实现用户认证功能

任务进度：
  总计：8 个任务点
  已完成：5 个
  未完成：3 个

最后提交：2026-03-20 15:30:45 +0800 (3天前)
```

### 场景：使用 -v 参数

**操作**：
```bash
aide hi -v
```

**预期输出**：
在精简输出基础上增加：
- git 仓库状态详情
- 配置文件路径
- plantuml 编译状态

## ADDED: aide go 命令

### 场景：无参数且仅一个未归档任务

**前置条件**：
- tasks/ 下仅有 task-3/
- 当前在常驻分支

**操作**：
```bash
aide go
```

**预期输出**：
```
提示：检测到仅有一个未归档任务，自动跳转
→ 切换到任务分支 task-3
✓ 已切换到分支 task-3
```

### 场景：指定任务编号切换

**前置条件**：
- tasks/ 下有 task-3/ 和 task-5/
- 当前在常驻分支

**操作**：
```bash
aide go 5
```

**预期输出**：
```
→ 切换到任务分支 task-5
✓ 已切换到分支 task-5
```

### 场景：仓库不干净时切换

**前置条件**：
- 有未暂存的变更
- auto_commit_on_switch = true

**操作**：
```bash
aide go 3
```

**预期输出**：
```
⚠ 检测到未暂存的变更
→ 自动暂存并提交
✓ 提交：暂存：清理仓库状态以切换分支
→ 切换到任务分支 task-3
✓ 已切换到分支 task-3
```

## ADDED: aide bye 命令

### 场景：在常驻分支执行

**前置条件**：
- 当前在常驻分支
- 仓库干净

**操作**：
```bash
aide bye
```

**预期输出**：
```
✓ 当前已在常驻分支 dev
✓ 仓库状态干净
再见！
```

### 场景：在任务分支执行

**前置条件**：
- 当前在任务分支 task-3
- 有未暂存变更

**操作**：
```bash
aide bye
```

**预期输出**：
```
→ 暂存并提交变更
✓ 提交：暂存：清理仓库状态
→ 切换到常驻分支 dev
✓ 已切换到分支 dev
再见！
```
