# 提案：aide 程序核心子命令（hi/go/bye）

## 概述

实现 aide 程序的三个核心子命令：hi（状态查询）、go（进入任务分支）、bye（清理并回到常驻分支）。

## 动机

新的 aide 体系需要简洁高效的命令来：
- 快速查看项目和任务状态（hi）
- 快速切换到任务工作分支（go）
- 快速清理并回到常驻分支（bye）

这三个命令是用户和 Agent 与 aide 体系交互的核心入口。

## 目标

1. 实现 `aide hi [-v]` 状态查询命令
2. 实现 `aide go [n] [-v]` 分支切换命令
3. 实现 `aide bye` 清理命令
4. 集成 plantuml 自动编译功能

## 非目标

- 不实现任务管理功能（verify/confirm/archive）
- 不实现阶段管理功能
- 不实现 memory 生成功能

## 设计

### aide hi [-v]

**功能**：查询并展示当前项目状态

**输出内容**：
- 项目目录绝对路径
- 当前 git 分支
- 根据分支类型显示不同信息：
  - 常驻分支：未归档任务列表 + 摘要 + 最后提交时间
  - 任务分支：当前任务摘要 + todo 统计 + 最后提交时间
  - 其他分支：提示信息

**参数**：
- `-v, --verbose`：显示详细信息（默认精简）

### aide go [n] [-v]

**功能**：切换到任务工作分支

**行为**：
- 无参数 + 仅一个未归档任务 → 自动跳转
- 无参数 + 多个未归档任务 → 显示提示
- 有参数 → 切换到指定任务分支
- 仓库不干净 → 根据配置自动暂存提交

**参数**：
- `n`：任务编号
- `-v, --verbose`：切换后显示详细状态

### aide bye

**功能**：清理仓库并回到常驻分支

**行为**：
- 在常驻分支：检查并清理仓库状态
- 在任务分支：暂存提交 + 切换到常驻分支
- 在其他分支：仅提示

## 实现计划

1. 实现 git 操作封装（分支查询、切换、提交）
2. 实现 branches.json 读取和解析
3. 实现 task-summary.md 读取（跨分支）
4. 实现 todo.md 解析（统计任务点）
5. 实现 plantuml 变更检测和编译
6. 实现三个子命令的 CLI 接口

## 测试计划

- 测试 go 的各种参数组合
- 测试 bye 的各种场景
- 测试 plantuml 自动编译
